# strictcli features available for adoption

This is optional. Use whatever benefits this project, ignore the rest. This todo is generic across all strictcli consumers.

## .strictcli/ directory

strictcli now auto-generates a `.strictcli/schema.json` file describing the full CLI structure (commands, groups, flags, args, types, defaults, help text). Run the CLI with `--dump-schema` to generate it.

This can be wired into the pre-release hook so the schema stays current on every release. The commit uses the `Autogenerated: true` trailer so it's exempt from changelog coverage.

To set up:
- Run `<cli> --dump-schema` to generate `.strictcli/schema.json`
- Create `.strictcli/config.json` with static metadata (language, entry point)
- Add schema generation to `.rlsbl/hooks/pre-release.sh`
- Commit both files

## Config file support

strictcli has built-in JSON config file support. When enabled with `App(config=True)` (Python) or `WithConfig()` (Go), the app:

- Reads `~/.config/{app_name}/config.json` (respects XDG_CONFIG_HOME)
- Uses precedence: CLI flag > env var > config file > code default
- Auto-registers `config show/set/path/edit` subcommands

Useful for flags that are set-once-and-forget (host, port, model, batch size).

## Auto-version from metadata (Python only)

`App(name="myapp", help="...")` without a version parameter auto-detects from `importlib.metadata`. No more stale hardcoded version strings after rlsbl bumps.

## type=float

Float flags are now supported: `Flag(name="rate", type=float, help="...")` (Python) or `FloatFlag("rate", "...", Default(1.0))` (Go). Rejects NaN and Inf.

## Recursive group nesting

Groups can contain subgroups to arbitrary depth: `app -> group -> subgroup -> command`. Useful for CLIs with service/resource/action hierarchies.

## --help anywhere

`--help` and `-h` are now recognized anywhere in the argument list, not just as the first argument after the command name. `cmd --verbose --help` now shows help.
